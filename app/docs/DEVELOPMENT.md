# Руководство по разработке

## Архитектура проекта

Crypto App использует модульную архитектуру с четким разделением ответственности:

### Структура директорий
- `lib/core/` - ядро криптографических алгоритмов
- `lib/models/` - модели данных
- `lib/services/` - бизнес-логика и сервисы
- `lib/screens/` - пользовательский интерфейс
- `tests/` - юнит-тесты

## Криптографические алгоритмы

### Кузнечик (ГОСТ Р 34.12-2015)
- Реализован в `lib/core/Kuznyechik/`
- Симметричный блочный шифр
- Размер блока: 128 бит
- Размер ключа: 256 бит

### Стрибог (ГОСТ Р 34.11-2012)
- Реализован в `lib/core/streebog/`
- Криптографическая хеш-функция
- Поддерживает выходы 256 и 512 бит

### RSA
- Реализован в `lib/core/rsa/`
- Асимметричное шифрование
- Используется для обмена ключами

## Добавление новых алгоритмов

1. Создайте модуль в `lib/core/`
2. Добавьте юнит-тесты в `tests/`
3. Обновите документацию
4. Создайте Pull Request

## Тестирование

```bash
# Все тесты
flutter test

# Конкретный тест
dart test tests/kuznechik_test.dart

# Покрытие кода
flutter test --coverage
```

## CI/CD

Проект использует GitHub Actions для:
- Автоматического тестирования
- Сборки для Android и Windows
- Генерации API документации
- Создания релизов

## Стандарты кода

- Используйте `dart format .` для форматирования
- Следуйте соглашениям именования Dart
- Добавляйте документационные комментарии
- Пишите тесты для новой функциональности
